Create and Host a Blog for Free (~1 hour)

Use GitHub Pages and Hugo to get your blog online in about an hour (at no cost).

 Posted on Sat January 1, 2022  |  7 minutes  |  1356 words  |  Denise Case

Thinking about sharing some interesting content on the web? Modern tools make it easy to publish content for free.

Choosing An Approach

Writers who don’t want to work directly in HTML, CSS, and JavaScript (three key technologies for building web sites) often chose a Content Management System (CMS) such as WordPress.

There’s another option - using a “static site generator”. For these, content is written in a simple markup language and then run through a process to create the HTML for a web page. Hugo is a static site generator written in Go.

These notes show how to use Hugo and GitHub to get your blog up and running in about an hour.

New to GitHub?

If you aren’t already using GitHub, create a free account on GitHub.

Choose a username and provide an email and password. Your username will become part of your hosted URL (the address of your website).

You can use your GitHub account to create respositories (typically called repos for short). Each repo is like a folder or directory that holds files for a particular project.

Create Two GitHub Repositories

First, create two GitHub repos, one private and one public:

  1. Create a new GitHub repo for developing content. Make it private with no files. This repo will hold the source for the website. I call mine notes-source, but blog-source or something similar will work great.
  2. Create a new GitHub repo for the auto-generated content. Make it public with no files. It will hold the files that will be automatically published using GitHub Pages. I call mine notes, but blog or similar would work as well.

Enable GitHub Pages in the Public Repo

Next, we’ll set up the public repo so that the content is automatically published with GitHub Pages. In your second (public) repo, click Settings. In the repo settings, scroll down to “GitHub Pages”, click “Check it out here!”. Change “None” to main, click “Save”.

It displays the URL for your free site. Mine is https://denisecase.github.io/notes/ - yours will be similar, but will show your username and the name of your public repo.

Install Tools

You’ll need Git and Go installed.

  • Git version control system. Configure Git with your username and email.
  • Go programming language.
  • VS Code for editing files (VS Code is recommended, not required). Be sure to turn on File / Auto Save, to save edits automatically so you never need to remember to save a file.

Installation Option: Package managers make it easy to install software and keep it current. On Windows, I use Chocolatey. Follow the linked instructions to install Chocolatey, then open PowerShell as Administrator and run the following commands to install:

choco install git -y
choco install golang -y
refreshenv
git --version
go version

Install Hugo

Install Hugo static site generator.

For example, on Windows using Chocolatey, run the following.

choco install hugo-extended -y -confirm
refreshenv
hugo version

Create a Source Repo on Your Local Machine

Use Hugo to generate a folder of content with the same name as your new private repo. My commands are as follows - adjust them to use your GitHub username instead of denisecase and your private repo name instead of notes-source.

hugo new site notes-source
cd notes-source
hugo mod init github.com/denisecase/notes-source
git init
git submodule add https://github.com/halogenica/beautifulhugo.git themes/beautifulhugo

Customize Hugo Configuration

To configure the website, we need to customize config.toml. I recommend copying the information from the example config.toml file into yours, then editing yours to reflect your URL, author information, and content.

Run Your Site Locally

hugo server

Open browser to http://localhost:1313/notes. Replace notes with the public repo name created at the beginning.

Once It Runs, Add More Content

Create additional files in your repo:

  • .gitignore
  • README.md
  • .github/workflows/deploy.yml
  • content/_index.md
  • content/post/2022-01-01-publish.md

Follow these general instructions:

  • The first file, .gitignore can be exactly the same as the linked example file.
  • Customize the README.md as you like - it’s an overview page for your project, and won’t appear in your hosted website. It’s a good place to keep track of necessary commands and links. It’s written in GitHub Markdown, an easy-to-use markup language.
  • .github/workflows/deploy.yml is the file that tells GitHub to run some tasks whenever we push new content up to our main branch. It will generate the website, and copy the content files to the public repo (which will then be automatically hosted with GitHub Pages). In your editor, find the line with external_repository: denisecase/notes and change it to use your GitHub username (instead of denisecase) and your public repo name (instead of notes). You’ll also need to replace ${{ secrets.PERSONAL_TOKEN }} with the actual generated token you create below.
  • content/_index.md defines the opening page of your blog - customize this as as you like.
  • You’ll need to change the file name and file contents of the last file content/post/2022-01-01-publish.md- this will become your first article. Follow all naming conventions and be careful when editing the contents. Errors can keep your site from running. If you get errors, just revert to the content in this repo and then slowly modify it while running. If the code breaks, you’ll know where to focus.

One Time: Create A GitHub Personal Access Token

In your .github/workflows/deploy.yml, you’ll see personal_token: ${{ secrets.PERSONAL_TOKEN }}. This gives our action authorization to write files into a different repository.

Create a GitHub Personal Access Token by following the instructions here. Set the token to never expire (or choose a different date) and check the top-level box for repo and admin:repo_hook which will select all their sub-items.

When you finish creating the token, you’ll get one chance to copy the token to your clipboard - do so. Copy the token to your clipboard and replace ${{ secrets.PERSONAL_TOKEN }} with your token.

In your private repo, the line should look something like this:

personal_token: ghp_abcdefghijklmnopq...

One Time: Link Your Local Private Source Repo to Your GitHub Source Repo

We’ve already run git init in the private repo. Now, change the default branch to main, set the origin alias to your GitHub repo URL, then git add all the files, git commit all the changes, and then git push up the cloud repo.

In the following commands, change denisecase to your GitHub username and notes-source to your private repo name.

git branch -M main
git remote add origin https://github.com/denisecase/notes-source.git
git add .
git commit -m "first commit"
git push -u origin main

Recurring: Watch Your Site As You Code

If your site is still running, you’ll see the changes in the browser as you make edits. If your site is not running (or if you get errors), edit the content then restart the site with hugo server and open browser to http://localhost:1313/notes. Remember to replace notes with your public repo name.

Recurring: After Adding New Content

After adding content, always do a git add to add any new files, then git commit with a message of what changed, and git push up to GitHub.

git add .
git commit -m "added new content"
git push -u origin main

View Your New Site on the Web

To see your new website, open a brower to the GitHub Pages URL shown when you added GitHub Pages to your public repo. For example, my URL is https://denisecase.github.io/notes/, but yours will use your GitHub username and your public repo name.

For convenience, you might add your URL to your private README.md and/or display it in the repo About section shown in the upper right when viewing the repo contents as shown here. (Click the cog icon on your public repo to update the meta information for your website.)
